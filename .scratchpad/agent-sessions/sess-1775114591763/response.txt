# SSO 登入討論 - 技術視角 (Agent 1)

## 核心優勢
1. **無縫體驗**：一次登入，多處通行，減少重複輸入密碼的疲勞感
2. **密碼管理簡化**：用戶只需記住一個主密碼，降低密碼洩漏風險
3. **集中式安全**：密碼變更、重置或 MFA 設定都在一個地方管理
4. **無縫整合**：透過 OIDC/SAML 協議，輕鬆串接現有目錄服務 (如 Azure AD, Okta)

## 技術挑戰
1. **依賴鏈複雜度**：主帳號一旦出問題，所有依賴它的系統都可能受影響
2. **權限粒度控制**：在單一身分源下，細微的權限管理有時會顯得不夠靈活
3. **延遲與同步**：用戶登入後，權限同步到各子系統可能需要幾秒鐘
4. **供應商鎖定**：過度依賴特定 SSO 供應商，切換成本較高

## 最佳實踐建議
- 採用 **SSO + 本地帳號** 的混合模式，保留退路
- 實施 **SAML 2.0 或 OIDC** 標準，確保長期兼容性
- 加入 **MFA 強制** 提升安全性，而不只是依賴密碼
- 設計 **斷線重連** 機制，處理 SSO 節點不穩的情況

## 適合場景
- 中大型企業內部系統串接
- SaaS 生態系整合 (如 Slack + Jira + GitHub)
- 需要快速擴張的團隊 (避免重複設定帳號)
